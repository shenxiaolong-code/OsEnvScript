
csh shell:
1、部分常用的内部变量：区别大小写 , 下面的所有变量都必须以$打头来引用
#       : $# 变量是命令行参数或位置参数的数量
_       : $_ 变量是一个特殊的变量，在shell开始运行时，变量的初始值为shell或其执行的shell脚本的绝对路径名，之后就是最近执行的命令的最后一个选项或参数等。
?       : $? 测试变量是否定义，结果值为1或者0，可用于if条件语句中,e.g. if () 。 类似： dos : if defined Var; windbg : ${/d:Var}
$       : $$ 变量是shell脚本里面的进程ID (pid)。Shell脚本经常使用 $$ 变量组织临时文件名，确保文件名的唯一性。
!       : &! 变量的值是最近运行的一个后台进程的PID
*       : $* 变量的值表示所有的位置参数，其值是所有位置参数的值。

经验证，csh脚本中不可用或者用法未知的变量：
-       : $- 变量是传递给shell脚本的执行标志
@       : $@ 变量的值类似于$*，表示所有的位置参数，其值也是所有位置参数的值。

!$      : PID of the last program your shell ran in the background (e.g. myprog &)

$status 或 $? 表示退出状态, 相当于window下的 %errorlevel%
$?prompt    //prompt是否已经启用，结果值为1或者0，可用于if表达式中。 也可用于测试当前是静默模式还是交互模式
$cwd/$PWD   : 当前工作目录。 cwd/.用于脚本中表示当前目录。 cwd是shell变量，可以用户指定。 PWD等同于cwd,但其不能用户指定。其仅仅在目录确切改变时才会被自动更新。 所以 PWD 更符合我们期望的意义。 
              小写的pwd是命令，输出当前目录绝对路径。 pwd -P输出当前目录的物理路径(解析符号链接)。

以下用法不可用或者用法未知：
RANDOM      : 随机数变量。每次引用这个变量会得到一个0~32767的随机数。    
OLDPWD      : 利用cd命令改换到新目录之前所在的工作目录。用法：cd $OLDPWD  （切换到之前的工作目录，和cd - 功能一样）
LINENO      : 调试用。用于显示脚本中当前执行的命令的行号。
OPTARG      : getopts命令已经处理的前一个选项参数。
REPLY       : read命令，如果没有被指定变量。即通过read读入的数据，如果没有对应的变量进行赋值，则可以把REPLY变量用作read命令的默认变量，接收read命令读入的参数。
SECONDS     : 脚本已经运行的时间（以秒为单位）


bash shell:
$$          Shell本身的PID（ProcessID）
$!          Shell最后运行的后台Process的PID
$?          最后运行的命令的结束代码（返回值）
$-          使用Set命令设定的Flag一览
$*          所有参数列表。如"$*"用「"」括起来的情况、以"$1 $2 … $n"的形式输出所有参数。
$@          所有参数列表。如"$@"用「"」括起来的情况、以"$1" "$2" … "$n" 的形式输出所有参数。
$#          添加到Shell的参数个数
$0          Shell本身的文件名
$1～$n      添加到Shell的各参数值。$1是第1参数、$2是第2参数…。
